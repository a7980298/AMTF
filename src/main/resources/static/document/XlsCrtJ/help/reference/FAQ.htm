ExcelCreator for Java
よくあるご質問

このトピックでは、ExcelCreator for Java のよくあるご質問について説明します。さらに詳しい内容は、弊社ホームページの「よくあるご質問」をご参照ください。

このトピックは、次のセクションを含みます。

  • 英語版など、日本語以外の環境に対応していますか。
  • xls 形式の Excel ファイルを使用できますか。
  • 体験版をアンインストールせずに、製品版をインストールしても良いでしょうか。
  • 体験版で作成したアプリケーションを製品版に置き換えて使用しても問題はないでしょうか。
  • 対応する Excel バージョン、ファイル形式を教えてください。
  • Excel ファイルで使用可能なデータの上限を教えてください。
  • マクロの編集や実行に対応していますか。

英語版など、日本語以外の環境に対応していますか。

ExcelCreator for Java は、日本語の環境のみ対応しています。英語版など、日本語以外の環境では動作を保証しておりません。

xls 形式の Excel ファイルを使用できますか。

ExcelCreator for Java は、xlsx 形式の Excel ファイルの扱いに対応しており、xls 形式の Excel ファイルは未対応です。

体験版をアンインストールせずに、製品版をインストールしても良いでしょうか。

体験版と製品版は共存することができません。そのため、ExcelCreator for Java の体験版を事前にアンインストールし、製品版をインストールしてください。

体験版で作成したアプリケーションを製品版に置き換えて使用しても問題はないでしょうか。

ExcelCreator for Java 体験版で作成したアプリケーションを、製品版に置き換えて使用することができます。JAR ファイルを差し替えてご対応ください。

対応する Excel バージョン、ファイル形式を教えてください。

ExcelCreator for Java は、以下の Excel バージョン、ファイル形式に対応しています。

Excel 2007 / 2010 / 2013 / 2016 / 2019 (xlsx 形式)

Excel ファイルで使用可能なデータの上限を教えてください。

Excel ファイルの制限は以下の通りになりますが、実際に設定可能な上限については、環境のメモリに依存します。

Excel ファイル (xlsx 形式) の制限

  • 行数 1,048,576 行
  • 列数 16,384 列
  • シート数 制限なし(実行環境の使用可能メモリに依存)

マクロの編集や実行に対応していますか。

ExcelCreator for Java は、VBA やマクロに関する部分の編集、および、実行には対応しておりません。
